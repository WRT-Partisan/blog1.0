Prim算法

类似于Dijkstra算法，时间复杂度：。

适用范围：稠密图。

做法

  1. 将号点加入当前生成树中并打上标记，同时更新数组。
  2. 选择所有未标记的点中最小的点。
  3. 将该点加入当前生成树中并打上标记。
  4. 使用与该点相连的所有连边更新数组。
  5. 重复第二步到第四步，直到所有的点都已经被标记或所有未标记的点都不存在连边与已标记的点直接相连。

模板题目

题目描述

给出一个无向图，求出最小生成树，如果该图不连通，则输出 orz。

输入格式

第一行包含两个整数，表示该图共有个结点和条无向边。

接下来行每行包含三个整数，表示有一条长度为的无向边连接结点。

输出格式

如果该图连通，则输出一个整数表示最小生成树的各边的长度之和。如果该图不连通则输出 orz。

Kruskal算法

时间复杂度：。（适用于稀疏图）

做法

  1. 将所有的边从小到大排序。
  2. 将所有点分别放入各自的并查集中。
  3. 选择所有未使用的边中边权最小的。
  4. 若该边所连接的两个点已经连通，则舍去，否则合并这两个并查集。
  5. 重复第三步到第四步，直到所有点都已经被标记或所有未标记的点都不存在连边与已标记的点直接相连。
